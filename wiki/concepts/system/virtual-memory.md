## Definition

虚拟内存是操作系统为每个进程提供的**独立、连续的虚拟地址空间抽象**。每个进程认为自己独占全部内存，实际物理内存由 OS 通过页表按需映射。

核心组件：虚拟地址空间 + 页表 + MMU + 缺页异常处理。

## 为什么需要虚拟内存

没有虚拟内存的世界：
```
进程A: 使用物理地址 0x1000~0x5000
进程B: 也编译成从 0x1000 开始 → 冲突
进程C: 需要 2GB 内存但机器只有 512MB → 无法运行
恶意进程: 直接读写物理地址 → 无隔离
```

虚拟内存解决三个问题：
1. **隔离**：每个进程独立地址空间，进程 A 的 0x1000 ≠ 进程 B 的 0x1000
2. **超卖（overcommit）**：进程可以使用超过物理内存的地址空间（通过 swap）
3. **保护**：页表项有 r/w/x 权限位，硬件强制检查

## 核心机制

### 地址转换流程

```
虚拟地址 (CPU 发出)
    │
    ▼
+-------+
|  MMU  |  ← 硬件单元，查 TLB
+-------+
    │
    ├── TLB hit → 直接得到物理地址 → 访问内存
    │
    └── TLB miss → 走页表遍历（page table walk）
            │
            ▼
        CR3 寄存器 → 四级页表（x86-64）
            │
            ├── present=1 → 物理页帧号 → 填入 TLB → 访问内存
            │
            └── present=0 → page fault → 内核处理
                              │
                              ├── 合法访问（已映射到磁盘/未分配）→ 分配物理页
                              └── 非法访问 → SIGSEGV
```

### x86-64 四级页表（48-bit 虚拟地址）

```
63    48 47    39 38    30 29    21 20    12 11        0
+--------+--------+--------+--------+--------+-----------+
| 未使用  | PML4   | PDPT   | PD     | PT     | offset    |
|        | 9 bits | 9 bits | 9 bits | 9 bits | 12 bits   |
+--------+--------+--------+--------+--------+-----------+
```

每级 512 项，每项 8 字节，每级页表刚好占一个 4KB 页。

### 页表项（PTE）结构

```
+-----+---+---+---+---+---+-----------+----------------+
| NX  | G | D | A | C | W | U | Rsvd    | 物理页帧号(40b)|
+-----+---+---+---+---+---+-----------+----------------+
  不可  全  脏  访  Cache 写  用户
  执行  局  位  问  策略  通  可访
             位       问
```

关键位：
- **P (present)**：0 触发 page fault
- **U (user)**：0 = 仅内核可访问
- **W (writable)**：0 = 只读，写触发 COW 或 SIGSEGV
- **A (accessed)**：硬件自动设 1，OS 用于 LRU 换页决策
- **D (dirty)**：硬件自动设 1，表示该页被写过，换出时需写回磁盘

## TLB（Translation Lookaside Buffer）

TLB 是 MMU 内部的硬件缓存，缓存虚拟→物理地址映射。典型大小：L1 TLB 64 项（数据）+ 64 项（指令），L2 TLB 512~2048 项。

```
TLB 在 4KB 页面下的覆盖范围：
L1 TLB 64 项 × 4KB = 256KB    ← 很小
L2 TLB 1536 项 × 4KB = 6MB    ← 仍然有限

大数据结构的 TLB miss 是性能杀手。
大页（2MB/1GB）可以让单条 TLB 项覆盖更多内存。
```

### TLB 与上下文切换

进程切换时 CR3 改变 → 所有 TLB 项失效（除非使用 PCID）。频繁上下文切换导致的 TLB miss（TLB shootdown）是重要的性能损耗来源。

## 关键概念

### Demand Paging（按需分页）

进程启动时不加载全部代码/数据。执行到某条指令时：
1. CPU 发出对应虚拟地址
2. MMU 查页表 → P 位 = 0 → page fault
3. 内核从磁盘加载对应页到物理内存 → 更新页表 → 返回用户态重试

好处：进程启动快，内存只给真正用到的部分。

### Swap（交换）

物理内存不足时，内核将不常访问的页面（通过 LRU 链表判断）写出到 swap 分区/文件，回收物理页。被换出的页再次访问时触发 page fault，从 swap 读回。

```
被换出页的 PTE：
P=0，其余位存储 swap 的位置信息（swap 分区号 + offset）
```

### 内存超卖（Overcommit）

Linux 默认允许进程申请超过物理内存的虚拟内存。因为进程通常不会用满所有申请的内存（稀疏使用）。只有当进程真正访问时才分配物理页。极端情况：所有进程同时访问所有申请的内存 → OOM Killer 介入。

```bash
# 查看 overcommit 策略
cat /proc/sys/vm/overcommit_memory
# 0: 启发式（默认）
# 1: 总是允许
# 2: 禁止超卖，最多分配物理内存 + swap 的比例
```
